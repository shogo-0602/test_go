# Go言語学習プロジェクト

> 最終更新: 2026年3月23日

---

## 📖 このプロジェクトについて

このプロジェクトは、プライベートでGO言語の学習を行ない、その過程を記録するものである。
基礎的な文法から順に実装を行い、各段階での学習成果を残していく。

---

## 📝 Commitメッセージのルール

本プロジェクトのコミットメッセージは、以下のル ール に従って記述する。

### メッセージ形式

```
<type>: <subject>

<body>
```

### 1. Type（タイプ）

以下のタイプから該当するものを選択する：

| タイプ | 説明 | 例 |
|--------|------|--------|
| `feat` | 新機能追加 | feat: ポインタの学習ページを追加 |
| `fix` | バグ修正 | fix: ループ処理のエラーを修正 |
| `docs` | ドキュメント変更 | docs: 初期化手順をREADMEに追加 |
| `refactor` | コード整理・改善（機能変更なし） | refactor: エラーハンドリング処理を簡潔に |
| `test` | テスト関連 | test: スライス操作のテストを追加 |
| `chore` | build, 依存関係、メタデータ更新等 | chore: go.mod を更新 |

### 2. Subject（主題）

- **1行70文字以内** で簡潔に記述する
- 日本語で記述する
- 命令形で記述する（〜した ではなく 〜する）
- 末尾に句点（。）を付けない

### 3. Body（本文）

- 変更の詳細や理由を記述する（必須ではないが、推奨）
- 複雑な変更の場合は記述する
- 改行で段落を区切る
- 日本語で記述する

### メッセージの例

```
feat: 条件分岐（if/else）のサンプルコードを追加

conditional_branch.go を新規作成し、if文とelse文の
基本的な使用例を実装。複数条件の組み合わせパターンも
含める。
```

```
fix: loop.go のループ終了条件を修正

forループが無限ループになっていたカウント変数の
更新処理を追加した。
```

```
docs: 環境構築手順をGO_README.mdに追加
```

### コミット時の注意点

- 1つの論理的な変更を1つのコミットで行う
- 複数の独立した変更は複数のコミットに分ける
- Subjectのみで説明できない場合は、Bodyで詳細を記述する
